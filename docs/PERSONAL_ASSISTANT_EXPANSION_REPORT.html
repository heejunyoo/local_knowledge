Section 0 · Operating principle

구현 가능한 최선 · 전략 대결 · 압축 P0

“표가 예뻐서 최선”이 아니다. 지금 코드·1인·로컬 스택에서 구현 가능한 최선만 P0다. 절대 최적 설계는 주장하지 않는다.

채택 합성. 주전략 = A 통합 Hub+Timeline. RAG 최소 신뢰(C)는 W0 동시 게이트. HealthKit(B)은 W1. Capture(D)는 W2. 에이전트/메일(E)은 OUT.

Section 1 · Scope

정의·경계

범위는 IN / EDGE / OUT 세 칸으로만 나눈다. 겹치지 않는다.

제품 정의. Mac Core가 데이터 주권·추론·오케스트레이션을 맡고, iPhone(+Watch→HealthKit 브리지)은 얇은 터치포인트로, 「지식 · 몸 · 하루」를 하나의 로컬 개인비서 표면에서 연결한다.
IN

Core Assistant

로컬 SoT 지식·신체·하루 통합, Tailscale Core, thin mobile. 2026-H2 포함.

EDGE

Optional later

영양 DB, 위젯, EventKit 읽기, 음성 인박스, HTTPS. W3+ 조건부.

OUT

Explicit non-scope

범용 에이전트, 메일, 소셜, Mac 직접 HK, 의료 진단, 멀티유저.
